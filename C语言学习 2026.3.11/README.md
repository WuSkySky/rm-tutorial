##### 结构体
- 结构体是什么?
    是一种自定义的含有多个基本类型的类型
    通过下面的语句可以自定义一种新的类型```struct Point ```
    ```C
    struct Point {
        int x;
        int y;
    };
    ```

- 结构体怎么使用
    通过上面的结构体类型定义```struct Point ```可以当作一个类型使用了,像这样
    ```C
    struct Point p1; // 变量声定义
    struct Point p2 = {3, 4}; // 同时进行变量定义与赋值,需按顺序
    struct Point p3 = {.y = 10, .x = 20}; // 同时进行变量声定义与赋值,无需按顺序
    p1 = {3, 4}; //这是错误的,{}的语法只用用在初始化时
    ```

- 成员访问运算符```.```
    通过成员访问运算符可以访问结构体变量的成员
    ```C
    struct Point p1 = {3, 4};;
    p1.x = p1.y;
    ```

#### 第三部分
##### 指针
其中一种理解方式
- 先引入一系列新的类型
    - 我们将引入一系列新的类型,指针类型,在之前类型的总数是有限的,但是在引入后类型的总数变成了无限个

    - 任意的类型都有一个指针类型与之对应,每个指针类型也有一个新的指针类型与之对应,所以是无限个

- 新类型的表示方法
    在一个类型后面加```*```,如
    ```C
    int* a; // 在int类型后加*
    float* b; // 在float类型后加*
    struct c* c; // 在struct c类型后加*
    int** d; // 在int*类型后加*
    int*** e; // 在int**类型后加*
    ```

- 再引入一个新的运算
    - 取地址运算, 是一元运算,在运算对象前加```&```表示这种运算,如```&a```

    - 可能猜到了,这个运算的返回类型就是指针类型,具体说返回类型是运算对象的所对应的指针类型

    - 举例来说,在这个运算```&a```中,如果```a```的类型是```int```,```&a```的类型就是```int*```

    - 再举一例,```&a```中如果```a```的类型是```int*```,```&a```的类型就是```int**```

    - 理所当然,取地址运算的结果可以赋值给对应的指针类型的变量如
        ```C
        int a;
        int*p;
        a = 10;
        p = &a;
        ```

    - 再来说取地址运算的含义,取地址运算的作用就是得到某个变量的内存地址

    - 应该能意识到,所以没有地址的东西不能作为地址运算的对象,下面这样是会报错的,因为```10```不是变量,没有确定,稳定的内存
        ```C
        int*p;
        a = &10;
        ```

- 再讨论一个容易疑惑的点,不同类型取地址后得到的都是内存地址,长度应该相同,为什么得到的变量类型不一样?
地址长度确实相同,但是变量类型除了包含了长度信息,还包含了,改怎么解释这个包含的内容的信息,所以类型会不同

- 再再引入一个新的运算
    - 解引用运算,引用在这里可以理解为指针,所以是"解指针运算"
    - 一元运算,表示方法是运算对象前加```*```,如```*a```
    - 解引用运算是取地址运算的逆运算,如果对象是```int*```类型,返回类型就是```int```,如果对象是```int**```类型,返回类型就是```int*```
    - 理所当然,无法对```int```类型的变量进行解引用运算

- 指针的加减运算

- 指针的下标运算```[]```
    可以理解为语法糖,不引入新特性

- ```->```运算
    - 这个运算的对象是指向结构体的指针
    - 这个运算还要指明访问的成员名
    - 含义是解引用加取成员
    - 表示方法是 指向结构体的指针->成员名
    - 如
        ```C
        struct Point
        {
            int x;
            int y;
        };
        struct Point a1 = {3, 4};
        struct Point *p = &a1;
        p->x = p->y;
        ```

##### 数组
- 引入一系列新类型
    - 我们引入一系列新类型,用于存储n个类型相同的数据,这就是数组
    - 可见数组有两个维度,类型和存储个数
    
- 表示方法
    - 这样表示```类型名[存储个数]```如```int[10]```
    - 注意定义数组类型的变量时,变量名要写在中间如```int a[10]```

- 定义与赋值
    - 定义
    ```C
    int a[5]; //定义
    ```
    - 初始化赋值
    ```C
    int a[5] = {1, 2, 3, 4, 5}; //定义并赋值
    a = {1, 2, 3, 4, 5}; // 是错误的,{}语法只能用于初始化
    ```

- 退化
    - 数组类型在很多场合下会退化为指向首个元素的指针
    - 最常见的情况是对数组类型用下标运算```[]```,这种情况很方便获取数组中的每个元素,如
        ```C
        int a[5] = {1, 2, 3, 4, 5};
        a[0] = a[1];
        ```

#### 第四部分
##### 多文件编程
- 构建
    构建是将包含多个源文件和头文件整个C语言项目变成可执行的二进制文件的过程,简单分为三个阶段
    - 预处理 
        - 输入是多个源文件和头文件
        - 进行纯文本替换
        - 产物有多个文件,被称为翻译单元
    - 编译 
        - 输入是多个翻译单元
        - 语法检查,进行二进制转化
        - 输出是多个二进制文件,称为目标文件
    - 链接
        - 输入是多个目标文件
        - 将他们合并在一起
        - 输出是一个二进制的可执行文件

- 全局变量
    - 定义的位置不在任何一个函数的内部
    - 运行时整个程序都可以访问,但编译时,只有该变量所在的翻译单元可见

- 变量的声明
    - 告诉编译器有某个变量,并告知类型
    - 语法```extern```加变量类型,加变量名,如 ```extern int a;```

- 编译会检查什么
    - 其中一个是检查变量有没有被声明或定义
    - 其中一个是检查变量被多次定义,也就是重定义

- 预处理指令
    - 先介绍一个预处理指令```#include```
    - 含义将这一句替换为某个文件
    - 用法
        - ```#include<文件路径>```在系统路径中寻找整个文件进行替换
        - ```#include"文件路径"```在系统路径中寻找整个文件进行替换

- 一般来说,头文件写什么
    - 想暴露的结构体类型声明
    - 想暴露的全局变量
    - 想暴露的函数声明,只写函数头就是函数声明
    
- 一般来说,源文件写什么
    - 头文件对应的全局变量的定义
    - 头文件对应的函数的定义
